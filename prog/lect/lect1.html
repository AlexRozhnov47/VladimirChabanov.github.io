Что такое программирование?

Программирование - процесс создания компьютерной программы для решения какой-либо практической задачи.

Этапы:

  1. Постановка задачи. Заключается в определении существенно важных объектов и (или) явлений предметной области, определении отношений между этими объектами и (или) явлениями, выделении и описании объекта исследования, исходных данных и результатов решения задачи;
  2. Формализация. Разработка формальной (математической) модели исследуемого объекта или явления;
  3. Создание технического задания на исходную задачу. Заключается в уточнении исходных данных, результата и способа получения результата из исходных данных, проводимого в рамках полученной математической модели.
  4. Разработка алгоритма решения задачи и его запись.
  5. Кодирование, тестирование, отладка и документирование программы.
  6. Получение и анализ результатов.

Понятие алгоритма

Под алгоритмом понимается заранее понятное и точное предписание возможному исполнителю совершить определенную последовательность действий для получения решения задач за конечное число дискретных шагов.

Свойства алгоритма

  • Дискретность (небольшие определенные шаги);
  • Определенность (четкость и однозначность, точная интерпретация команд);
  • Результативность (конечность);
  • Массовость (применим к целому классу подобных задач, различающихся только исходными кодами).

Формы представления алгоритмов

  • Словесная;
  • Графическая;
  • Псевдокод;
  • Программная.

Почему С++

  1. Высокий уровень сложности;
  2. Мультипарадигменность;
  3. Множество языков имеет Си подобный синтаксис.

Согласно TIOBI индексу C++ расположен на 4 месте. Индекс строится основываясь на количестве запросов в поисковых системах.

Краткие сведения о языке

Разработчик: Бьёрн Страуструп. Год создания языка: 1980 г.

Компилируемый, статически типизированный язык программирования общего назначения.

На данный момент актуальный стандарт: С++17. Разрабатывается черновик С++20. Наиболее распространён: С++11/С++14.

Подготовка рабочего места

  1. Скачать Visual Studio Community или альтернативную среду разработки;
  2. В процесс подготовки к установке проверить включены ли компоненты поддержки С++;

Что входит в состав Visual Studio Community

  1. Компилятор. Программа, преобразующая исходный код с какого-либо языка программирования на машинный язык;
  2. Редактор кода;
  3. Менеджер проектов;
  4. Инструменты отладки, профилирования и диагностики.

Минимальная программа на С++

Исходные коды программы на С++ содержатся в текстовых файлах с расширением .cpp, .cxx, ...

Данная программа не делает ничего полезного, она запускается и сразу же завершается.

Рассмотрим подробнее код.

В третьей строке расположен заголовок функции main. Это главная функция программы, то есть именно с 3й строки программа начнёт исполняться. Инструкции выполняются последовательно, строка за строкой и как только очередь дойдёт до команды return программа завершится, даже если после return ещё есть код.

Оператор return имеет один параметр, в данном случае это код ошибки с которым завершилась программа.

Код 0 означает, что программа завершилась без ошибок. Этот код сам по себе ничего не означает, это просто число, то есть для каждой программы можно самому придумать таблицу соответствия кода и ошибки. Код ошибки часто используется при написании терминальных скриптов. Об этом можно почитать например здесь.

По стандарту для функции main можно не указывать оператор return, в этом случае программа также завершится с кодом ошибки 0, но только после того, как будет выполнен весь код внутри тела функции main.

Тело функции main и любой другой ограниченно фигурными скобочками. Строка 4 - начало функции, строка 7 - конец функции (аналог begin и end в Паскале).

В 1й и 5й стоках можно увидеть две подряд идущие косые черты: //. Это признак начала однострочного комментария, то есть любой текст расположенный после этой пары символов и до конца строки полностью игнорируется.

Добавим программе возможность выводить сообщения в консоль. Для этого воспользуемся директивой препроцессора include следующим образом

Директивы препроцессора можно опознать по символу # следом за которой, без пробела, идёт название директивы.

Данную строку кода можно прочитать так: заменить эту строку на текст из файла с именем iostream. Препроцессор будет искать файл в директориях переданных компилятору с помощью параметра командной строки (-I), в путях заданных переменными окружения, в путях вшитых в компилятор. Обычно таким образом подключают стандартные библиотеки, поставляемые вместе с компилятором.

Другой вариант записи директивы include:

Библиотека iostream определяет три стандартных потока:

  • cin стандартный поток ввода. Можно применить для считывания введённых пользователем данных;
  • cout стандартный выходной поток. Позволяет выводить в консоль сообщения.
  • cerr стандартный поток вывода сообщений об ошибках. Практически тоже что и cout, разница в особенностях реализации.

Чтобы использовать стандартные потоки вывода нужно воспользоваться оператором <<.

xxxxxxxxxx
2
1
cout << "Сообщение";
2
cerr << "Сообщение";

Справа от оператора << расположено то, что нужно отобразить в консоле. Сообщением может быть: текст (в двойных кавычках), целое число, вещественное число и т.д.

Визуально оператор напоминает стрелочки указывающие направление движения информации, в данном случае сообщение (справа) отправляется в поток вывода. Написать так: "Сообщение" >> cout; нельзя.

Для потока ввода используется похожий оператор >>.

xxxxxxxxxx
2
1
int i;    // Переменная целого типа. Сюда будем считывать
2
cin >> i;

Чтобы получить информацию из потока ввода нужно иметь возможность её куда-то положить. В примере выше, всё что считается из потока попадёт в переменную i.

Здесь направление стрелочек так же можно воспринимать как направление движения информации.

Если нужно вывести или считать несколько раз подряд, можно воспользоваться сокращённым способом:

xxxxxxxxxx
2
1
cout << "Курс доллара: " << 67.72 << " рубля";
2
// На экране будет:  Курс доллара: 67.72 рубля

Во всех примерах потоки вызывались по сокращённому имени. Если попробовать это сделать в программе, то вы получите сообщение об ошибке. Устранить ошибку можно двумя способами:

  1. Каждый раз использовать полное имя потоков дописывая слева std::, например: std::cout.
  2. Выше функции main добавить строку using namespace std;
